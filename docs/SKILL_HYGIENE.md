# Skill Hygiene Scorecard

Generated by `scripts/build-catalog-enrich.py`. **The mean hygiene score is a structural signal** (frontmatter, description, length vs. progressive disclosure) — it is *not* a claim about whether the skill produces correct econometrics. For correctness see [`benchmark/`](../benchmark/) (numeric recovery) and [`eval-harness/`](../eval-harness/) (behavioural evals).

## Why two columns?

A skill can be **well-formed** (high hygiene) yet **untested for the traps an applied economist needs it to catch** (zero eval coverage), and vice-versa. This scorecard reports both so reviewers don't conflate them.

| Column | What it measures | Source |
|---|---|---|
| **Hygiene score** | Frontmatter, description, line count, `references/` presence | `scripts/build-catalog-enrich.py` |
| **Eval coverage** | How many [`eval-harness/scenarios/`](../eval-harness/scenarios/) target this skill | `eval-harness/scenarios/*.toml` → `skill = …` field |
