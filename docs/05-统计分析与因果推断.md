## CoPaper.AI 内置 20 个方法论 Skills

这是目前最完整的面向经济学实证研究的 Skill 套件，分为四大类：

### 基础回归方法（6 个）

| Skill | 适用场景 | 核心流程 |
|-------|---------|---------|
| **OLS 回归** | 最基础的线性回归 | 渐进式多规格对比 → 异方差检验 → 稳健标准误 |
| **Logit / Probit** | 二值因变量（是/否、成功/失败） | 模型选择 → 边际效应计算 → 拟合优度 |
| **Tobit 截断回归** | 因变量有大量零值（研发投入、捐赠金额） | 截断点设定 → 估计 → 边际效应 |
| **面板数据分析** | 固定效应 / 随机效应 | Hausman 检验 → 模型选择 → 聚类标准误 |
| **GMM 动态面板** | 因变量含滞后项 | Arellano-Bond / Blundell-Bond → AR 检验 → Sargan 检验 |
| **描述性统计** | Table 1 + 相关系数矩阵 | 变量描述 → 均值差异检验 → VIF 共线性诊断 |

### 因果推断方法（6 个）

| Skill | 适用场景 | 核心流程 |
|-------|---------|---------|
| **DID 双重差分** | 政策评估的经典方法 | 平行趋势检验 → 基准回归 → 4 项稳健性检验 → 异质性分析 → 机制分析 |
| **交错 DID** | 不同个体在不同时间受处理 | Callaway-Sant'Anna / Sun-Abraham / de Chaisemartin 新方法 |
| **IV 工具变量** | 内生性问题 | 第一阶段回归 → 弱工具检验 → 2SLS → 过度识别检验 |
| **RDD 断点回归** | 利用阈值处的不连续性 | 带宽选择 → 局部多项式估计 → 密度检验 → 协变量平衡 |
| **PSM 倾向得分匹配** | 构造可比对照组 | Logit 估计 → 匹配 → 平衡性检验 → ATT 估计 |
| **合成控制法** | 单一处理单元的政策评估 | 权重优化 → 拟合度检验 → 安慰剂检验 |

### 前沿方法 + 机制分析（6 个）

| Skill | 适用场景 | 核心流程 |
|-------|---------|---------|
| **双重机器学习 (DML)** | ML 灵活控制混淆变量 | 交叉拟合 → 残差化 → ATE/CATE 估计 → 异质性分析 |
| **ML 预测建模** | 分类/回归预测 | 多模型对比 → 超参调优 → 交叉验证 → 特征重要性 |
| **中介效应** | 因果机制检验（X → M → Y） | Baron-Kenny → Sobel 检验 → Bootstrap 置信区间 |
| **异质性分析** | 分组回归 + 交互项 | 子样本分析 → 交互项回归 → 组间系数差异检验 |
| **Heckman 选择模型** | 样本选择偏差修正 | 选择方程 → 修正回归 → Lambda 显著性判断 |
| **时间序列 / GARCH** | 波动率建模 | 平稳性检验 → ARIMA → GARCH → 条件波动率 |

### 专项分析流程（2 个）

| Skill | 适用场景 | 核心流程 |
|-------|---------|---------|
| **事件研究法** | 金融事件 + 政策事件 | 事件窗口 → 异常收益率(CAR) / 动态效应 → 显著性检验 |
| **稳健性检验通用模板** | 适用于所有回归方法 | 6 大类 17 项检验清单 |

### 每个 Skill 内部结构

不是一段笼统的提示词，而是**结构化的分步操作流程**：

```yaml
---
name: did-analysis
description: 双重差分法因果推断完整分析流程
category: methodology
target_agent: modeling
tags: [causal-inference, DID, panel-data]
---

## 前置条件
- 面板数据，包含个体ID、时间、处理组标识、处理时间
- 结果变量已定义

## Step 1: 平行趋势检验
- 绘制处理组与对照组的结果变量时间趋势图
- 回归检验：处理前各期的交互项系数是否联合为零

## Step 2: 基准回归
- 渐进式多规格：(1)无控制变量 (2)加控制变量 (3)加固定效应
- 使用 save_table() 输出回归表

## Step 3: 稳健性检验（至少4项）
- 安慰剂检验
- 替换因变量
- 缩尾/截尾处理
- 排除特定样本

## Step 4: 异质性分析
...

## Step 5: 机制分析
...
```

## 实用建议

1. **不要跳过描述性统计**：Table 1 是审稿人看的第一张表。用描述性统计 Skill 确保你的变量分布合理、样本量充足
2. **稳健性不是走过场**：6 大类 17 项稳健性检验清单，至少做 4 项。如果核心结论在多种检验下都稳健，审稿人就没什么好挑的
3. **新方法用新工具**：如果你的数据是交错处理（staggered treatment），务必用 Callaway-Sant'Anna 等新方法，不要用传统 TWFE——它有严重偏差
4. **DML 不是万能的**：双重机器学习的优势在于灵活性，但它需要足够大的样本量。小样本下传统参数方法可能更可靠
