# AERS 2026-09 月度工作规划（让证据链可被外部 agent 接走）

> 制定日期：2026-07-04。8 月把"声称"升级到"端到端证据"：Card–Krueger 复现 PERFECT、
> 公开 scoreboard 上线（reference 16/16 vs naive 0/16 任务）、bunching 方法族闭环、
> 链检 triage 自动化、Card 1995 IV 复现案例入套。9 月主线：**把以上证据对外开放出去，
> 让外部 agent 能在三十分钟内接走这套评分体系**。

- [ ] 在 `Show and tell` 分类发置顶帖（含手把手）——任何 agent 跑同 16 个任务，发 JSON 即
      被 SCOREBOARD 收录；提交门槛=可复现（候选 JSON + 怎样跑出 + 一条命令）。
- [ ] `scripts/collect-external-scoreboard.py`：扫描 `benchmark/candidates/external-*/results.json`，
      渲染 `docs/EXTERNAL_SCOREBOARD.md`（同 `BENCHMARK_SCOREBOARD.md` 风格），加进
      `make catalog`，README/落地页挂入口。
- [ ] 仓库的 `SCOREBOARD` 规则文档化（`docs/SCOREBOARD_RULES.md`）：谁能提交、评分怎么算、
      如何避免人为挑任务、duplicate 合并规则。

**完成判据**：第一条非自研 candidate（即使是 AERS 的"naive baseline"，也算独立提交
的示范）出现在 EXTERNAL_SCOREBOARD；外链可以一键访问。

## 第 3 周（09-15 ~ 09-21）：可安装的 CLI —— 30 秒安装即评分

- [ ] 抽出 `aers-score` CLI 工具：单一 Python 文件（仅 stdlib），`aers-score --candidate
      path/to/candidate.json --task <id>` 直接出三档命中率 + 与 reference 的差距。
      包成 `pipx install` 友好的最小分发（pyproject.toml + `aers_score` console script）。
- [ ] `pip install -e .` 可工作；CI 在 push 时构建并上传 artifact（不走 PyPI，
      走 release asset；不增加维护成本）。

**完成判据**：外部 agent 作者在 30 分钟内从 README 走到本机 CLI 出自己的成绩单，
过程全在 README 里能拼出来。
