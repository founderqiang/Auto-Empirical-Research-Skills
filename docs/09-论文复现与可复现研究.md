## 实战案例：10 分钟复现一篇 AER 论文

这不是假设，而是国内社区的真实操作记录。

**论文**：《Canal Closure and Social Unrest: Evidence from Qing China》（运河废弃与社会动荡：来自清代中国的证据），发表在 American Economic Review。

**操作**：用 Claude Code 的复现 Skill，输入一条指令"帮我复现这篇论文的结果"，**10 分钟**自动生成完整复现报告。

### 复现详情

| 维度 | 说明 |
|------|------|
| **设计** | DID，6 省 536 县，1650-1911 面板，1826 年运河废弃为处理事件 |
| **翻译** | 原始 Stata `.do` 文件自动翻译为 Python `pyfixest.feols()` |
| **覆盖** | 漕运量趋势（图 2）、平行趋势（表 2）、基准 DID（表 3）、事件研究（图 4）、南北差异（表 5） |

### 核心结果对比

| 图表 | 状态 | 关键数字 |
|------|------|---------|
| 图 2（漕运量） | 完全复现 | 1826 年后下降约 15% |
| 表 2（平行趋势） | 完全复现 | 处理前趋势系数 0.0003，p=0.59，通过检验 |
| 表 3（基准 DID） | 核心一致 | 系数 0.042~0.061，方程 1-3 显著 |
| 图 4（事件研究） | 基本复现 | 处理后 +20 期系数 0.0613** |
| 表 5（南北差异） | 完全复现 | 北方三重交互系数 0.10~0.12*** |

**整体复现评级**：B 级（核心系数方向和显著性与原文高度一致）。

> 以前复现一篇 AER 可能需要好几天。现在一句话，10 分钟。**当你能把"复现一篇顶刊"这种最硬核的任务自动化时，"跑一个稳健性检验"或"换一种识别策略试试"就更不在话下了。**

## 可复现研究最佳实践

### 项目结构

```
project/
├── data/
│   ├── raw/          # 原始数据（不修改）
│   └── processed/    # 清洗后的数据
├── code/
│   ├── 01-clean.py   # 数据清洗
│   ├── 02-analysis.py # 主分析
│   └── 03-robustness.py # 稳健性
├── output/
│   ├── tables/       # 回归表格
│   └── figures/      # 图形
├── paper/
│   └── draft.tex     # 论文稿件
├── SKILL.md          # Agent Skills 配置
├── requirements.txt  # Python 依赖
└── README.md         # 说明文档
```

### 五条原则

1. **版本控制**：用 Git 管理代码和数据处理流程
2. **环境锁定**：Python 用 `requirements.txt`，R 用 `renv`，Stata 用 `version` 命令
3. **数据不可变**：原始数据放 `data/raw/`，永不修改
4. **代码即文档**：每个脚本开头注明输入、输出和目的
5. **一键复现**：提供 `make` 或 shell 脚本，一个命令跑完全部分析

---

## 实用建议

1. **Git 不是可选项**：你不需要精通 Git，但你需要让 Agent 能用 Git。Git 提供安全网和版本历史，让你安全地尝试和回滚
2. **复现包是投稿标配**：越来越多的期刊要求提交复现包。在研究初期就按照上面的项目结构组织文件，投稿时省很多事
3. **Skill 本身就是复现文档**：一个好的 SKILL.md 不仅是给 AI 的操作手册，也是给人类读的方法论文档
