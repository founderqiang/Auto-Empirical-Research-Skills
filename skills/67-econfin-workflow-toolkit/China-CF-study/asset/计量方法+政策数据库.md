## 1. 速查总表（方法 × 政策 × 数据）

### 1.1 按方法找政策（Method-First）

| 方法 | 适合的政策类型 | 中国典型案例 | 优先微数据 |
|---|---|---|---|
| **Sharp RDD** | 有清晰得分/年龄/边界阈值的政策 | 高考分数线、法定退休 60 岁 | CFPS / CHARLS |
| **Fuzzy RDD** | 阈值非完全决定治疗的政策 | 新农保资格、低保门槛 | CHARLS / CFPS |

## 2. RDD（断点回归设计）

### 2.1 六种 RDD 变体

| 变体 | 关键特征 | 适用情景 | 中国常见例 |
|---|---|---|---|
| **Sharp RDD（清晰）** | 阈值 100% 决定处理 | 法律/制度刚性阈值 | 高考一本分数线、法定退休 60 岁 |
| **Fuzzy RDD（模糊）** | 阈值仅提升处理概率 | 资格类政策 | 新农保 60 岁、低保收入门槛 |
| **Spatial RDD（地理）** | 边界两侧条件突变 | 区域性政策 | 秦岭-淮河供暖线 |
| **RDiT（时间）** | 单一时点切换 | 全国性即时生效政策 | 一带一路 2013、营改增 |
| **Multi-cutoff** | 多阈值序列分类 | 等级评定类政策 | 信用评分、AQI 等级、淘宝店铺等级 |
| **Donut RDD** | 剔除阈值附近样本 | 操纵/堆积明显时 | 利润 0 附近的盈余管理 |

### 2.4 RDD 使用陷阱与稳健性

1. **操纵性检验**（McCrary density test）：被处理者能否操纵阈值附近的位置？若分数线附近样本分布有显著跃迁，识别失效。
2. **带宽选择**：用 IK / CCT 最优带宽 + 多带宽稳健性。Stata 用 `rdrobust`，Python 用 `rdrobust-py`。
3. **协变量平衡**：阈值两侧的非处理变量分布应连续。报告"covariate balance test"。
4. **多项式阶数**：报告 1 阶、2 阶、3 阶多项式拟合结果。审稿人通常不接受 4 阶以上。
5. **Donut hole**：剔除阈值附近 ±0.1 或 ±1 单位样本验证。
6. **安慰剂阈值**：把阈值人为移动到非真断点处（如 50 分、200 分），系数应不显著。
7. **Spatial RDD 专属**：报告"距边界距离"作为运行变量；考虑 latitude × longitude 双向 RDD。
8. **RDiT 专属**：必须报告时间趋势检验（pre-trend in pre-period）。

## 3. DID（双重差分）

### 3.1 五种 DID 变体

| 变体 | 关键特征 | 代码骨架 |
|---|---|---|
| **经典 2×2 DID** | 单一时点 + 单一处理组 | OLS with treat × post |
| **Staggered DID（TWFE）** | 多时点逐步推开 | 双向固定效应 + treat × post |

| **Sun-Abraham IW** | Interaction-weighted estimator | `pyfixest sunab()` |
| **Stacked DID** | 每个 cohort 单独 stack | 数据 reshape + TWFE |

### 3.5 DID 常见陷阱

1. **TWFE 负权重**：staggered DID 用 TWFE 时存在负权重，需用 CS-DID / Sun-Abraham 验证。
2. **平行趋势**：必报 pre-treatment 期系数；失败时按 Plan-B 重定义时点。
3. **预期效应**：政策"公布"与"实施"年份的区分；预期效应导致 pre-trend。
4. **同期政策混淆**：检索同时期是否有其他政策干扰；加入 co-shock dummy。
5. **聚类**：标准误聚类层级 = 处理变异层级（如政策在省层面变异，聚类到省）。
6. **样本选择**：金融业、ST、IPO 当年默认剔除；缩尾 1%/99%。
7. **安慰剂**：1000 次随机分配处理组 + 随机抽政策时点，真实系数应在虚假分布外。

## 4. DDD（三重差分）

### 4.1 何时用 DDD 而非 DID

DDD 在以下三种情形优于 DID：

1. **政策对子群体有异质化处理强度** — 比如绿色金融试点对重污染企业冲击远超清洁企业，第三维度（污染密度）使设计更精确；
2. **需要排除同期混杂政策** — 同时期其他政策可能影响处理组与对照组的"差异变化"，第三维度（如不暴露于该混杂政策的子集）可以排除；
3. **政策强度连续变化** — 政策实施强度（如转移支付比例、税额标准）在地区间存在连续差异，DDD 可识别"剂量-反应"关系。

### 4.2 DDD 三维度常见组合

| # | 时间 | 第二维 | 第三维 | 典型应用 |
|---|---|---|---|---|
| 1 | 政策时点 | 试点地区 | 企业污染密度 | 绿色金融、环保税、碳交易 |
| 2 | 政策时点 | 试点地区 | 企业规模 / 融资约束 | 减税降费、信贷政策 |
| 3 | 政策时点 | 试点地区 | 企业国有 / 民营 | 混改、产业政策 |
| 4 | 政策时点 | 国家 / 行业 | 行业暴露度 | 贸易协定、FTA |
| 5 | 政策时点 | 地区发展度 | 个体特征（性别 / 教育） | 教育改革、生育政策 |
| 6 | 政策时点 | 试点地区 | 地理邻近性 | 自贸区、基础设施 |
| 7 | 政策时点 | 地区财政压力 | 政府行为类型 | 营改增财政压力 |
| 8 | 政策时点 | 行业技术水平 | 行业数字化深度 | 数字经济、AI 试点 |

## 5. PSM / PSM-DID / 熵平衡

### 5.1 何时用 PSM

- 试点地区与对照地区**前期不可比**（如自贸区 vs 内陆城市）；
- 样本量足够大（处理组 + 对照组各 ≥ 500）；
- 政策不是随机分配，存在选择偏差。

### 5.2 三种匹配方法对比

| 方法 | 优点 | 缺点 | 中国应用情景 |
|---|---|---|---|
| **PSM（最近邻 1:k）** | 简单直观 | 共同支撑域损失大；维度灾难 | 上市公司样本 |
| **熵平衡（EB）** | 同时平衡均值 / 方差 / 偏度；不丢样本 | 计算开销大 | 微观调查（CFPS、CHARLS） |
| **CEM（粗化精确匹配）** | 直观；保留 ATE 加权 | 高维变量易丢样本 | 行业 / 地区双匹配 |

### 5.3 PSM-DID 标准流程

```
Step 1：定义处理组与对照组（如试点城市 vs 非试点城市）
Step 2：用 pre-treatment 期协变量 Logit/Probit 估计倾向得分
Step 3：1:k 最近邻匹配 + caliper（如 0.002）
Step 4：检验匹配前后协变量平衡：|% bias| 应 < 5%
Step 5：在匹配样本上跑 DID：y_it = α + β·treat × post + FE + ε
Step 6：稳健性 — 替换 k=1,3,5；替换 caliper；改用 Kernel matching / 熵平衡
```

### 5.4 PSM-DID 中国典型应用

1. 自贸区设立 × 上市公司绩效 / 创新
2. 国家创新型城市 × 企业创新产出
3. 长护险试点 × 老年人福利
4. 国家级新区 × 企业投资 / 产业升级
5. 高新技术企业认定 × 企业税负 / R&D
6. 撤县设区 × 房价 / 财政
7. 国家自主创新示范区 × 城市创新
8. 沪港通标的 × 公司治理
9. 服务外包示范城市 × 企业服务化
10. 跨境电商综合试验区 × 中小企业出口
11. 双随机一公开 × 企业合规
12. 国家大数据综合试验区 × 数字化转型

### 5.5 平衡性检验图表（必出）

- **PSM 平衡前后条形图**：横轴 |% bias|，纵轴变量名；匹配前与匹配后双色条

### 6.3 IV 工程要点（中国情境）

- **第一阶段 F 统计量 > 10**（Stock-Yogo 弱 IV 阈值）；多 IV 时报 Kleibergen-Paap rk Wald F。
- **过度识别检验**（Hansen J / Sargan）：多 IV 时必报。
- **简化型回归**：直接看 IV 与 Y 的关系（reduced form），方向应与结构式一致。
- **外生性论证**三条路径：
  1. 时间分离（历史变量 vs 当代 Y）
  2. 机制特定（IV 只能通过 X 影响 Y）
  3. 制度隔离（IV 改变制度环境，不直接干预微观决策）
- **稳健性**：替换 IV、子样本检验、控制可能的混杂路径。

### 6.4 Bartik / Shift-Share 标准写法

```
IV_it = Σ_j (s_{ij,0} × Δ_jt)
其中 s_{ij,0}：地区 i 行业 j 的基期份额（外生）
     Δ_jt：行业 j 在 t 期的全国冲击（外生）
```

中国情境改造：

- 进口竞争：基期出口结构 × 全球行业进口变化（中国冲击）
- 数字经济：基期通讯设备投资份额 × 全国数字化指数
- 绿色金融：基期重污染行业份额 × 全国绿色信贷扩张

## 7. SCM（合成控制法）

### 7.1 何时用 SCM 优于 DID

| 情景 | 优先 SCM | 优先 DID |
|---|---|---|
| 处理单位数 | 1-3 个 | 多个 |
| pre-trend 平行性 | 不平行 | 平行 |
| 处理时点对齐 | 任意 | 对齐或差异小 |
| 可视化需求 | 强（轨迹图） | 弱 |
| 安慰剂检验 | In-space / in-time | 1000 次随机分配 |

### 7.2 SCM 标准流程

```
Step 1：选择捐助池（donor pool）— 与处理单位前期可比的非处理单位
Step 2：用预政策期协变量 + 滞后 Y 求最优权重 W
Step 3：构造合成对照 = Σ w_i × (non-treated unit i)
Step 4：政策后对比处理单位 Y(actual) vs Y(synthetic)
Step 5：In-space placebo — 把每个 donor 当作处理跑一次，看 RMSPE 排名
Step 6：In-time placebo — 把政策时点提前 5 年，验证 pre-trend
Step 7：稳健性 — 排除单一权重过大的 donor；改变 donor pool
```

### 7.4 SCM 扩展方法

- **Augmented SCM**（Ben-Michael et al. 2021）：用 ridge 回归修正合成对照与真实 pre-trend 的偏差
- **Penalized SCM**：在权重求解中加 L1 / L2 惩罚，避免单一 donor 主导
- **Synthetic DID**（Arkhangelsky et al. 2021）：SCM + DID 双重去偏；适合 staggered 设计

## 8. Spatial DID（空间双重差分）

### 8.1 何时用 Spatial DID

- 政策有**地理溢出效应**（如高铁开通、自贸区设立）
- 处理组与对照组**空间相邻**（违反 SUTVA）
- 关心**直接效应 + 间接效应**分解

### 8.2 空间权重矩阵选择

| 矩阵类型 | 设定 | 适用 |
|---|---|---|
| **0-1 邻接（Queen）** | 相邻为 1 | 县级 / 地级市分析 |
| **逆距离** | 1/d | 长距离溢出（高铁、贸易） |
| **k-近邻** | 最近 k 个为 1 | 处理稀疏地区 |
| **经济距离** | 1/(GDP_i − GDP_j)^2 | 经济联系强度 |
| **二元贸易** | 实际贸易量 | 行业链溢出 |

### 8.3 模型形式

```
SAR（空间滞后）  ：  y = ρWy + Xβ + ε
SLX（空间外生）  ：  y = Xβ + WXθ + ε
SDM（空间杜宾）  ：  y = ρWy + Xβ + WXθ + ε  ← 推荐
SDID（动态）    ：  y_it = ρW y_{it} + λW y_{i,t-1} + ...
```

### 8.4 中国 Spatial DID 应用（12+）

1. 数字经济大数据综合试验区 → 新质生产力（城市群溢出）
2. 低碳城市试点 → 制造业高质量发展（正 / 负溢出）
3. 国家超算中心 → 周边城市能源效率
4. 高铁建设 → 城市碳排放
5. 自贸区 → 大气污染（邻近地区溢出）
6. 绿色金融试点 → 区域碳减排
7. 主体功能区战略 → 碳排放格局
8. 制度型开放 → 区域协调发展
9. 长三角一体化 → 城市产业升级
10. 京津冀协同 → 河北产业溢出
11. 国家自主创新示范区 → 邻近城市创新
12. 数字基础设施 → 城市出口韧性

## 9. DML（双重机器学习）

### 9.1 何时用 DML

- **控制变量维度高**（如行业 × 地区 × 时间细分类）
- **非线性混杂关系**（传统 OLS / FE 难以建模）
- **中等样本量**（N ≥ 1000，DML 在小样本表现差）

### 9.2 DML 三大变体

| 变体 | 适用 | 关键学习器 |
|---|---|---|
| **DML-PLR** | 连续 X，无内生 | LASSO、随机森林、LightGBM |
| **DML-IV** | X 内生 | 同上 + IV |
| **Causal Forest / GRF** | 异质性处理效应 | 因果森林 |

### 9.3 DML 标准实现（Python doubleml）

```python
from doubleml import DoubleMLPLR, DoubleMLData
from sklearn.linear_model import LassoCV
from sklearn.ensemble import RandomForestRegressor

dml_data = DoubleMLData(df, y_col='y', d_cols='x',
                        x_cols=controls)
model = DoubleMLPLR(dml_data,
                     ml_l=RandomForestRegressor(n_estimators=500),
                     ml_m=LassoCV(),
                     n_folds=5, n_rep=10)
model.fit()
print(model.coef, model.se, model.pval)
```

### 9.4 中国 DML 应用（10+）

1. 数据要素市场化 → 企业创新（DML 控制高维财务变量）
2. 审计政策 → 财政失效（DDML）
3. 政府补贴 → 绿色创新
4. 低碳城市政策 → 企业出口（多期 DID + DML）
5. 自贸区 → 企业出口升级
6. 数字经济政策 → 企业创新
7. 高铁建设 → 空间关联（Spatial DID + DML）
8. 新质生产力政策评估
9. 人工智能创新试验区 → 企业全要素
10. 绿色金融 → 企业绿色转型

### 9.5 DML 边界

- **不适合纯因果识别（无 quasi-experiment 时）**：DML 假设可观测控制变量足以满足 unconfoundedness，仍是 selection-on-observables。
- **变量重要性 ≠ 因果**：feature importance 是预测重要性，不是因果重要性。
- **必须报告交叉拟合的 fold 数**与 reps（如 5 折 × 10 reps）。

## 10. 门槛回归 / Bunching

### 10.1 Hansen 面板门槛模型

适合**X 与 Y 关系存在非线性折点**的情景（如普惠金融发展水平、能源消费强度）。

```
y_it = β_0 + β_1 X_it · 1(q_it ≤ γ) + β_2 X_it · 1(q_it > γ) + Z_it + u_it
其中 q_it 为门槛变量，γ 为待估门槛值
```

### 10.2 Bunching 估计

适合**X 在某个阈值附近有"堆积"现象**（如所得税起征点、政策资格线）：

- 用阈值附近的密度分布偏离平滑趋势的程度估计行为反应弹性
- 中国应用：研发强度阈值高新技术认定、企业规模阈值（小微 vs 一般）、个税专项扣除门槛

### 10.3 中国应用（10+）

2. 能源消费与经济增长非线性关系

## 11. 事件研究 / Stacked / CS-DID / Sun-Abraham

### 11.1 事件研究规格

```
y_it = α_i + λ_t + Σ_{k=-K, k≠-1}^K β_k · 1(t - T_i = k) + ε_it
其中 T_i 为单位 i 的处理时间，β_k 为相对处理时间 k 期的动态效应
```

- 基准期通常设 k = -1
- 报告"动态效应图"：横轴 k，纵轴 β_k + 95% CI
- pre-period（k < 0）系数应不显著（平行趋势）

### 11.2 Callaway-Sant'Anna（CS-DID）

- 按"队列"（cohort = 同一年开始处理的单位）分别估计 ATT(g, t)
- 再按所需权重聚合
- Python: `differences` 包；Stata: `csdid`

### 11.3 Sun-Abraham（Interaction-Weighted）

- 等同于"k × cohort"完全饱和的交互项 OLS
- 用 last-treated 或 never-treated 作为对照
- Stata: `eventstudyinteract`；Python: `pyfixest.feols("y ~ sunab(cohort, year) | ...")`

### 11.4 Stacked DID

- 对每个 cohort g，构造"局部数据集"：g 队列 + never-treated（或更晚处理的）
- 各 cohort 局部数据集垂直堆叠
- 在堆叠数据上跑 TWFE
- 优点：避免 TWFE 负权重；缺点：堆叠数据样本量爆炸

### 11.5 不同估计量对比图（必出）

```
┌─────────┬───────┬───────────┬─────────────┬──────────┐
│  方法   │ TWFE  │   CS-DID  │ Sun-Abraham │ Stacked  │
├─────────┼───────┼───────────┼─────────────┼──────────┤
│ 系数    │ +0.02 │   +0.05   │   +0.04     │  +0.05   │
│ SE      │ 0.01  │   0.02    │   0.02      │  0.02    │
│ 95% CI  │ ±0.02 │   ±0.04   │   ±0.04     │  ±0.04   │
└─────────┴───────┴───────────┴─────────────┴──────────┘
```

四种估计量结论方向一致 → 主结论稳健；TWFE 与其他三者相反 → 存在负权重问题，以 CS-DID 为准。

## 附录 C：补充：少见但重要的方法 / 政策

### C.1 少见但应知的方法（PDF 中提及但本 md 未单列章节）

- **Heckman 两步法**：处理样本选择偏差（如上市公司样本对全体企业的代表性）
- **Tobit / 截尾模型**：因变量在 0 处堆积（如出口企业 vs 不出口）
- **Heckit + IV 双重纠正**：选择偏差 + 内生性
- **Maximum Entropy（ME）**：极度小样本 + 有限信息
- **断点 DiD（RDD × DiD）**：在 RDD 设计上叠加时间 DID
- **Donut RDD**：剔除阈值近邻样本（应对操纵）
- **Geographic Multiple Cutoffs**：多边界 RDD（多省份接壤地区）
- **Spatial Bayesian Models**：空间贝叶斯估计
- **GMM 动态面板**（Arellano-Bond / Blundell-Bond）
- **PVAR**（面板向量自回归）
