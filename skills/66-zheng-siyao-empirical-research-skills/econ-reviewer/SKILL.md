# econ-reviewer — 经济学顶刊标准审稿Skill

> 本 Skill 模拟**顶刊（AER, QJE, Econometrica, JPE, REStud）**匿名审稿人视角，对论文的**识别策略、实证结果、图表质量、理论机制**进行全面审查。**输出潜在致命缺陷列表和修改建议**，不替代研究者做决定。

## 适用场景

- 论文投稿前的自查（模拟审稿人视角）
- 与 Agent 协作写完论文后的独立审查（新开窗口，不同上下文）
- 审稿人意见回复前的预审
- 评估已发表顶刊论文的贡献时使用

## 核心理念

**不审格式，只审实质**。顶刊审稿的核心判断标准是：
1. **识别策略是否可靠**——因果推断是否干净
2. **理论机制是否有文献支撑**——不是拍脑袋
3. **结果是否稳健**——是否经过多种 robustness checks
4. **贡献是否明确**——相对于现有文献的增量贡献

**审稿不是挑刺，是帮助作者提升论文质量**。本 Skill 发现问题时必须给出具体的修改方向，而非仅指出"有问题"。

## 审查框架：六维审稿法

### 🔬 第一维：识别策略（Identification）

**审查要点**：

| 问题 | 严重度 | 审查标准 |
|------|--------|----------|
| 内生性问题 | 致命 | 工具变量/断点回归/面板固定效应等是否合理 |
| 选择偏差 | 致命 | 处理组和对照组是否可比 |
| 干扰因素 | 高 | 是否有遗漏变量偏误 |
| 外部效度 | 中 | 结果是否可推广到其他情境 |

**追问模式**：
> "作者使用了 XX 方法识别因果，请指出：
> 1. 该识别策略的核心假设是什么？
> 2. 该假设在本文情境下是否成立？
> 3. 若假设不成立，结果的方向是什么（有偏向上还是向下）？"

### 📐 第二维：理论机制（Mechanism）

**审查要点**：

| 问题 | 严重度 | 审查标准 |
|------|--------|----------|
| 机制不清晰 | 致命 | 理论机制是否有文献支撑 |
| 机制与结果脱节 | 高 | 机制分析是否与主结果逻辑一致 |
| 中介变量缺失 | 高 | 关键中介变量是否被检验 |
| 异质性分析不足 | 中 | 是否探索了效应异质性来源 |

**追问模式**：
> "本文声称 XX 机制是主要机制，请指出：
> 1. 该机制在文献中有何支撑？
> 2. 本文是否有直接的机制检验（如中介效应）？
> 3. 是否有排除其他机制的证据？"

### 📊 第三维：实证结果（Results）

**审查要点**：

| 检查项 | 标准 | 通过条件 |
|--------|------|----------|
| 基准回归系数 | 符号、大小、经济意义 | 可解释 |
| 标准误 | 聚类层面选择 | 与数据生成过程匹配 |
| R² | 解释力 | 不作为主要判断标准 |
| 效应量 | 经济显著性 | 与文献可比 |
| p值 | 统计显著性 | 多重假设检验校正 |

**追问模式**：
> "表2的主结果（列1）请审查：
> 1. 系数的经济意义是什么？
> 2. 标准误聚类在什么层面？是否正确？
> 3. 是否做了多重假设检验校正（如Bonferroni校正）？"

### 📈 第四维：图表质量（Figures & Tables）

**审查要点**：

| 图表类型 | 审查标准 |
|----------|----------|
| 描述统计表 | 变量定义清晰、样本量标注、N/A处理 |
| 回归表 | 系数可读、标准误标注、固定效应明确、控制变量列表 |
| 事件研究图 | 置信区间、零线、政策时点标注、样本量 |
| 并行趋势图 | 处理组/对照组区分、预趋势、零线 |

**追问模式**：
> "图1（事件研究图）请审查：
> 1. 政策前期的预趋势是否平行（系数是否显著异于0）？
> 2. 置信区间是否过宽（样本量是否足够）？
> 3. 政策后期效应是否单调或符合预期？"

### 🔒 第五维：稳健性（Robustness）

**必须检查的稳健性检验**：

| 检验类型 | 最低要求 |
|----------|----------|
| 替换自变量/因变量 | 至少1种替代指标 |
| 改变样本范围 | 排除异常值/特殊时期 |
| 改变控制变量组合 | 增减关键控制变量 |
| 改变模型设定 | 线性vs对数、固定效应层数 |
| 不同数据来源 | 替换数据库验证 |

**追问模式**：
> "本文报告了哪些稳健性检验？请指出：
> 1. 是否有"毁灭性"稳健性检验（即结果消失）？
> 2. 安慰剂检验的设计是否合理？
> 3. 结果对极端值、样本选择是否敏感？"

### 🎯 第六维：贡献声明（Contribution）

**审查要点**：

| 问题 | 严重度 | 审查标准 |
|------|--------|----------|
| 贡献不明确 | 致命 | 相对于文献的增量是什么 |
| 边际贡献 | 高 | 是否仅复制了已有发现 |
| 重要性 | 中 | 结果的政策或学术含义 |
| 写作质量 | 低 | 清晰度、逻辑性（最后考虑） |

**追问模式**：
> "本文声称的贡献是XX，请指出：
> 1. 该贡献在文献中的位置（补充、修正、还是颠覆）？
> 2. 现有文献为何没有得出类似结论？
> 3. 该贡献是否值得顶刊发表？"

## 输出标准格式

```markdown
## 顶刊标准审稿报告

**审查论文**：[论文标题]
**审查时间**：[日期]
**使用的模型/数据**：[简述]
**审查者视角**：AER / QJE / Econometrica / JPE / REStud

---

### 致命缺陷（必须修改，否则大概率拒稿）

1. **[缺陷1描述]**
   - 位置：表/图/正文第X段
   - 问题：...
   - 修改建议：...

2. **[缺陷2描述]**
   - ...

---

### 高严重度问题（修改后可能录稿）

1. ...
2. ...

---

### 中等严重度问题（建议修改，可选）

1. ...
2. ...

---

### 低严重度问题（写作/格式层面）

1. ...

---

### 总体评价

**识别策略**：可靠 / 基本可靠 / 存在问题 / 不可靠
**理论机制**：清晰 / 基本清晰 / 不清晰
**实证结果**：稳健 / 基本稳健 / 存在疑问
**顶刊适合度**：强烈推荐 / 推荐 / 修改后重投 / 不适合

**建议**：...
```

## 调用接口

在 Claude Code 对话窗口输入：

```
/econ-reviewer
```

或完整 Prompt：

```
按经济学顶刊标准（AER/QJE/Econometrica/JPE）审查本项目的论文输出（图表+回归表），识别策略、理论机制、实证结果、稳健性、贡献声明，列出所有潜在缺陷并给出修改建议。
```

## 与其他 Skill 的配合

- **`did-reviewer`** → DID 相关论文需配合 DID 专项审查
- **`citation-fidelity`** → 引用核查时配合使用

## 重要声明

- 本 Skill 审查的是**实证论文**，不适用于纯理论或方法论文
- **不替代正式审稿**，仅作为自查工具
- 论文最终是否投稿由研究者自行决定，文责自负
