# grillme — 苏格拉底诘问式选题Skill

> 本 Skill 模拟苏格拉底式诘问，通过连续追问帮助你逐步聚焦研究问题、识别研究空白、形成具有顶刊潜力的论文选题。**人在循环中注入判断，Agent 通过结构化提问引导方向。**

## 适用场景

- 研究方向不明确，需要逐步厘清
- 有初步想法但不知道如何聚焦成具体研究问题 (RQ)
- 想要验证自己的选题是否有新意
- 需要反直觉的视角来突破常规思维
- 对话窗口"不停对话讨论"时使用

## 核心理念

**不是给答案，而是通过提问逼你思考**。传统 ChatBot 是一问一答给结果；本 Skill 是苏格拉底诘问——Agent 持续追问，你选择方向，Agent 再追问，形成收敛思维。

**关键原则**：
1. **不直接给选题**——通过提问让你自己找到方向
2. **新意优先**——每个方向都要反问"这个角度是否已被做过？"
3. **可验收**——最终选题必须满足：① 有明确研究问题 ② 有可操作的变量 ③ 有理论机制 ④ 有新意

## 工作流程

```
初始陈述 → 追问聚焦 → 反直觉角度生成 → 机制推演 → 收敛确认
```

### Step 1 · 初始陈述

你描述一个模糊的研究兴趣，例如：
> "我想做一个数字技术与不平等相关的经济学论文"

Agent 不直接回应，而是开始系统化追问。

### Step 2 · 连续追问（聚焦子领域）

Agent 从以下维度连续追问，每次等你回复后再推进：

| 追问维度 | 问题示例 |
|----------|----------|
| **研究层次** | 你关注国家层面、企业层面还是个人层面？ |
| **不平等类型** | 收入不平等？机会不平等？数字鸿沟？制度差距？ |
| **因果方向** | 你认为技术扩散是加剧还是缓解不平等？ |
| **地理范围** | OECD 国家？发展中国家？特定地区？ |
| **时间维度** | 截面数据还是面板数据？关注哪个时间段？ |
| **数据类型** | 主观感知不平等还是客观指标？ |

### Step 3 · 反直觉角度生成

当你选定一个方向后，Agent 给出 **三个反直觉的研究角度**，例如：

- **角度 A（数字扩大制度差距）**：技术扩散顺序本身通过制度路径影响发展差距，而非传统收入渠道
- **角度 B（数字鸿沟双刃剑）**：互联网普及对不平等的影响呈倒 U 型——初期扩大、后期收敛
- **角度 C（技术扩散顺序诅咒）**：后发国家接受新技术时存在"路径依赖陷阱"，加剧长期不平等

### Step 4 · 机制推演

选择角度后，Agent 帮你推演理论机制：
1. **识别机制**：技术扩散通过什么渠道影响不平等？（直接效应 / 溢出效应 / 制度中介）
2. **提出假设**：基于机制提出可检验的研究假设
3. **数据匹配**：该假设需要什么数据来验证？（世界银行WDI / 中国CHIP / …）

### Step 5 · 收敛确认

最终输出格式化的研究问题文档：

```markdown
## 研究问题 (RQ)

**主标题**：数字技术扩散顺序对OECD国家发展差距的影响——基于制度路径的分析

**子问题1**：技术扩散顺序是否通过制度质量渠道影响国家间发展差距？

**子问题2**：数字鸿沟的倒U型效应在什么发展阶段最显著？

**理论机制**：技术扩散 → 制度质量 → 发展差距（制度中介）

**新意声明**：现有文献多关注技术对收入的直接效应，本研究首次从制度路径分析技术扩散顺序的发展影响。
```

## 调用接口

在 Claude Code 对话窗口输入：

```
/grillme
```

或完整 Prompt：

```
/grillme 基于本数据（OECD国家1990-2020年数字技术与社会发展指标）想一个有顶刊潜力的经济学论文选题，不断问我问题帮我厘清思路，要有新意未被做过
```

## 与其他 Skill 的配合

- **先用 `/grillme`** 厘清研究问题和方向
- **再用 `did-reviewer`** 设计实证策略（若涉及 DID）
- **用 `citation-fidelity`** 核查文献中的理论机制
- **用 `econ-reviewer`** 事后审查完整论文

## 停止条件

本 Skill **必须满足以下条件才可停止**：
1. 输出了明确的研究问题 (RQ)
2. 明确了理论机制路径
3. 确认了新意（文献中未被做过）
4. 你（研究者）明确选择了某个角度并确认

**未满足以上条件不得停止，不得直接给出一个模糊选题让你接受。**

## 重要声明

本 Skill 是**辅助厘清思路的工具**，不是替代研究者做决定。最终的研究问题文责自负，Agent 帮你结构化思考，但判断权在你手中。
