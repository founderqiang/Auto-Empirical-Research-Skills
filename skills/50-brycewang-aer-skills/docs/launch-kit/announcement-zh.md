## 知乎（文章/回答两用）

**标题备选：**
1. 我把"AI 写经济学论文"的每一个环节都装上了机器验证的门，开源了
2. 为什么大部分 "AI 学术写作工具" 过不了 AER 初审？我做了一个识别优先的开源方案
3. 从选题到 R&R 回复：一个被 CI 强制执行质量门的经济学论文 agent skill 栈

**正文骨架：**

AI 辅助写实证论文最大的风险不是写不出来，而是三种"看起来对"：引用是编的、数字是漂的、估计量是手搓错的。AER-Skills 的做法是把顶刊审稿人会检查的东西前移成机器门，且这些门先在仓库自己身上强制执行：

- **引用 groundedness 门**：每条散文引用必须解析到经 Crossref 验证的 bib 条目，幻觉引用（PHANTOM_CITATION）直接构建失败；
- **数值正确性契约**：每个方法演示（DiD、IV、RDD、合成控制、shift-share、DML、bunching、QTE、LP-DiD、矩阵补全、随机化推断……）都把估计值钉到已知真值 ± 容差，"跑通但答错"就是红灯；
- **工具绑定契约**：方法 skill 只能路由到已验证的工具注册表，不允许现场手搓估计量；
- **验证工具自身有单元测试**，全部门在 CI 上每次 push 强制执行。

覆盖 15 个 skill：选题 → 文献定位 → 识别策略 → 稳健性 → 正文 → 引言 → 表图 → 全稿一致性审计 → 对抗性模拟审稿 → 复现包 → 投稿 → R&R 回复。有一篇虚构论文走完全部 12 步的端到端 walkthrough。

GitHub：https://github.com/brycewang-stanford/AER-Skills （MIT，中英双语文档）
